# Entry, starters, and input cards

## Entry model

Use entry to collect required context before free-form messaging.

Typical fields:

- audience
- use case
- region

## Starter packs

Starter packs provide curated first interactions.

Best practices:

- keep labels short
- cap items per pack
- hide after first interaction

## Structured input cards

Use input cards for reusable mini-forms in chat.

Supported by schema-driven fields and submit behavior.

Keep card semantics generic in core; company-specific meaning belongs in consumer layers.
